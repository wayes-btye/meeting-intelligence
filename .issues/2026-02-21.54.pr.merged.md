### `frontend/app/chat/page.tsx`

**Markdown rendering (lines 157–159)**

```tsx
<div className="prose prose-sm max-w-none dark:prose-invert [&>*:first-child]:mt-0">
  <ReactMarkdown remarkPlugins={[remarkGfm]}>{result.answer}</ReactMarkdown>
</div>
```

- `[&>*:first-child]:mt-0` is a good touch — removes the top margin `prose` injects, avoiding a visual gap inside the card. 
- No XSS risk: `react-markdown` strips raw HTML by default. This is safer than `dangerouslySetInnerHTML`.
- `whitespace-pre-wrap` removal is correct — Markdown parsing handles linebreaks via blank lines/double-space, so the old pre-wrap would have interfered.
- `remarkGfm` adds tables, strikethrough, and task lists — all reasonable for Claude responses.

**One worth watching:** `react-markdown` v10 is ESM-only. Next.js 14 transpiles ESM deps, so the Vercel build passing confirms it works. But if you ever hit "must use import to load ES module" errors in tests or edge cases, pinning to `^9.0.0` is the escape hatch.

---

**Null speaker fallback (lines 173–177)**

```tsx
{source.speaker ? (
  <Badge variant="secondary">{source.speaker}</Badge>
) : (
  <Badge variant="outline" className="text-muted-foreground">Unknown speaker</Badge>
)}
```

- Correct guard — previous `<Badge variant="outline">{source.speaker}</Badge>` would have rendered an empty badge for null.
- The variant swap (`outline` → `secondary` for known speakers) is a minor visual change that gives named speakers more visual weight. Fine as-is.

---

**Pre-existing issue (not introduced here):** `key={i}` on source cards at line 170. Array-index keys are harmless here since sources don't reorder, but worth noting for when this component grows.
