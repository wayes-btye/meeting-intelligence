## Design notes
- Chunks section uses `<details>/<summary>` (no new component imports)
- Chunk index badge uses array position (`i`) since the API returns chunks pre-ordered by `chunk_index` but does not include `chunk_index` in the `SourceChunk` response shape
- Both sections are collapsed by default so the extracted items area stays clean
- 384-px max-height with overflow scroll for large meetings
