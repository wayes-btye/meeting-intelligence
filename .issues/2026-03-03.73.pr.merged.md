### P1 — Migration must precede deployment

**File:** `src/ingestion/storage.py:42`

`store_meeting()` unconditionally includes `chunking_strategy` in every insert payload:

```python
{
    ...
    "chunking_strategy": chunking_strategy,  # always sent
}
```

The migration file is written but **not applied**. This repo deploys to Cloud Run on every push to `main` (via Cloud Build trigger). If this PR merges before `supabase db push --linked` is run, **every ingest call** — including zip batch ingestion — will immediately fail with a PostgREST `"column does not exist"` error.

The migration itself is well-written (`IF NOT EXISTS` guard makes it idempotent), but the deploy ordering is load-bearing here. **Apply the migration before merging**, not after.

Codex flagged a backward-compatible code guard as an alternative — omit the key from the insert dict when the value is `None` (which would silently no-op on un-migrated DBs). That's valid too, but cleaner to just apply the migration first given this project's single-DB setup.

[Fix this →](https://claude.ai/code?q=Apply%20migration%2020260303121138_add_chunking_strategy_to_meetings.sql%20via%20%60supabase%20db%20push%20--linked%60%20from%20the%20main%20workspace%20before%20merging%20PR%20%2373%20on%20branch%20feat%2F48-chunking-strategy.%20The%20migration%20file%20is%20at%20supabase%2Fmigrations%2F20260303121138_add_chunking_strategy_to_meetings.sql&repo=wayes-btye/meeting-intelligence)

### Minor — `MeetingDetail` TypeScript interface missing `chunking_strategy`

**File:** `frontend/lib/api.ts:71-80`

`Meeting` (line 14) received `chunking_strategy?: string | null` — but `MeetingDetail` didn't:

```ts
export interface MeetingDetail {
  id: string;
  title: string;
  created_at: string;
```

The backend `MeetingDetail` Pydantic model does expose it (`src/api/models.py:68`), so the data is present in the API response — TypeScript just can't see it. This means `detail.chunking_strategy` can't be surfaced in the `MeetingDetailPanel` without a type error, if that's ever wanted.
